# Test Vectors

PNG carriers + expected plaintext, generated by the shipping Swift engine.
The Python reference decoder verifies that an independent implementation
written from the spec alone produces the same plaintext.

## Layout

- `manifest.json` — index of vectors with mode, password, expected plaintext.
- `*.png`         — carriers, one per vector.

## Regenerate

The vectors are not stable byte-for-byte (stealth mode uses random salt /
nonce / padding), so they are committed once and only refreshed when the
wire format changes.

```bash
xcodebuild test \
    -project Stiger.xcodeproj -scheme Stiger \
    -destination 'platform=iOS Simulator,name=iPhone 17' \
    -only-testing:StigerTests/SpecTestVectorsTests
```

Source: [`../../StigerTests/SpecTestVectorsTests.swift`](../../StigerTests/SpecTestVectorsTests.swift).

## Verify

```bash
cd ../reference/python
pip install -r requirements.txt
python verify_test_vectors.py ../../test-vectors
```

A passing run is the proof of "no backdoor": the bytes produced by the
Swift binary on your phone decode under a 250-line Python script that
nobody at our shop has ever touched at runtime.
